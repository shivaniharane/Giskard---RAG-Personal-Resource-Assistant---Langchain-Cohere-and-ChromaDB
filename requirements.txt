# ============================================================
# NOTES FOR DEVELOPERS
# ============================================================
#
# 1. COHERE MODEL VERSIONS (CRITICAL)
#    command-r-plus was PERMANENTLY REMOVED on September 15, 2025.
#    Always use dated versions:
#      command-r-08-2024         → standard, recommended
#      command-r-plus-08-2024   → more capable, higher token cost
#      command-r7b-12-2024      → lightweight, saves trial quota
#      command-a-03-2025        → latest flagship
#    Check current models: https://docs.cohere.com/docs/models#command
#
# 2. COHERE TRIAL KEY LIMITS
#    Free trial keys are limited to 20 calls/month on trial endpoints.
#    The Giskard scan calls chain.invoke() for EVERY generated test case.
#    A 5-category Giskard scan may generate 50-150 test invocations.
#    To avoid hitting limits:
#      a) Use command-r7b-12-2024 (fewer tokens per call)
#      b) Scan only 1-2 categories at a time: only=["hallucination"]
#      c) Upgrade to a paid Cohere key for full scans
#
# 3. TWO API KEYS REQUIRED
#    COHERE_API_KEY: for embeddings (CohereEmbeddings) + generation (ChatCohere)
#    OPENAI_API_KEY: for Giskard's internal scan engine (GPT-4 generates tests)
#    Without OPENAI_API_KEY, giskard.scan() will fail.
#
# 4. CHROMADB PERSISTENCE IN COLAB
#    /content/chroma_db is DELETED when Colab session disconnects.
#    For persistence: mount Google Drive and set:
#      persist_directory = "/content/drive/MyDrive/chroma_rag_db"
#    First run: indexes all PDFs (slow, costs Cohere API calls)
#    Subsequent runs: loads instantly from Drive (no re-embedding)
#
# 5. GISKARD REPORT MUST BE SAVED BEFORE SESSION ENDS
#    report.to_html("giskard_report.html") saves to /content/ (ephemeral)
#    Always run IMMEDIATELY after scan:
#      from google.colab import files
#      files.download("giskard_report.html")
#    OR copy to Drive:
#      !cp giskard_report.html "/content/drive/MyDrive/"
#
# 6. raise_exceptions=True STOPS SCAN ON FIRST ERROR
#    If any test case raises a Python exception, the entire scan aborts.
#    For more resilient scanning (especially with trial key rate limits):
#      giskard.scan(model, dataset, raise_exceptions=False)
#
# 7. CHUNK SIZE GUIDANCE FOR RESEARCH PAPERS
#    chunk_size=2048, chunk_overlap=512 (used in this project):
#      → Good for dense academic text with long paragraphs
#    For papers with many equations/formulas:
#      chunk_size=1024, chunk_overlap=256 → better precision
#    For papers with tables:
#      Consider unstructured library for better table extraction
#
# 8. DEFAULT RETRIEVER RETURNS k=4 CHUNKS
#    vectordb.as_retriever() retrieves 4 chunks by default.
#    For complex multi-part questions:
#      vectordb.as_retriever(search_kwargs={"k": 6})
#    For focused single-topic questions:
#      vectordb.as_retriever(search_kwargs={"k": 2})
